Error Handling

Websockets are inherently asynchronous, so error handling can be inflexible.

SimpleWebsockets.jl offers the user event hooks to register callbacks and handle errors flexibly.

SEE: Server Events, Client Events, Connection Events

If no callbacks are registered, hardcoded @info, @warn and @error calls will provide logging feedback, but the parent process will remain unaffected by exceptions.

WebsocketError

SimpleWebsockets.WebsocketError — Type
abstract type WebsocketError <: Exception

WebsocketError child error types have the following fields:

  • msg::String
  • log::Function

The log() function will internally call: @error msg exception = (err, trace)

Where trace is the backtrace of the exception origin.

Convenience methods

SimpleWebsockets.logWSerror — Function
logWSerror(err::Union{Exception, WebsocketError})

A convenience method to lift errors into a scope and log them.

Example

using SimpleWebsockets

server = WebsocketServer()
ended = Condition()

listen(server, :client) do client end
listen(server, :connectError) do err::WebsocketError
    notify(ended, err)
end

@async serve(server, 8080, "notahost")

reason = wait(ended)
reason isa Exception && logWSerror(reason)
